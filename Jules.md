# Genesis Webリニューアルプロジェクト：最終チェック＆改善仕様書（Jules向け）

このドキュメントは、Genesis合同会社のコーポレートサイト・リニューアルプロジェクトにおける、最終段階のリファクタリング、バグ検知、セキュリティチェック、および不要なテストコードの削除をJulesに依頼するための仕様書です。

## 1. プロジェクト概要
- **使用技術**: Astro, Tailwind CSS, TypeScript
- **デプロイ先**: Cloudflare Pages（想定）
- **主要機能**: 企業情報、各事業部（トータルソリューション、ライバーマネジメント等）の紹介、Resendを用いたお問い合わせフォーム

## 2. Julesへの依頼事項

### 2.1 テストコードと不要ファイルの削除
開発用に作成したプレビュー画面やダミーデータは、本番環境では不要かつセキュリティリスクとなるため削除してください。
- `src/pages/preview-email.astro` を削除する。
- その他、開発中に残された `console.log` やコメントアウトされた不要な古いコードをクリーンアップする。

### 2.2 コードのリファクタリング
- **Astroコンポーネント**: `src/pages/` やレイアウトファイルにおいて、重複しているロジックやHTML構造があれば、再利用可能なコンポーネントとして切り出す。
- **Tailwind CSS**: クラス指定が長すぎる箇所や、不要に重複しているスタイリングを整理する。
- **TypeScript**: `src/emails/ContactTemplate.ts` や `src/pages/api/contact.ts` の型定義をより厳格化し、`any` 型を可能な限り排除する。

### 2.3 セキュリティチェック＆堅牢性の向上
- **APIエンドポイントの保護**: `src/pages/api/contact.ts` に対して、不正なリクエストやスパム送信を防ぐためのバリデーション強化、またはレートリミットの考慮（Cloudflare側での設定提案など）を行う。
- **環境変数の管理**: `import.meta.env` で読み込んでいる `RESEND_API_KEY` などの機密情報がクライアントサイドに漏洩していないかを徹底的に監査する。
- **CORS / ヘッダー**: APIルートのレスポンスヘッダーにおいて、セキュリティベストプラクティス（適切なCORS設定など）が適用されているか確認する。

### 2.4 バグ検知とパフォーマンス改善
- **リンク切れチェック**: `src/pages/index.astro` などから遷移する内部・外部リンクにデッドリンクがないか検証する。
- **画像最適化**: 現在 `<img>` タグで読み込まれている画像（`assets/images/pages/` 等）について、Astroの `<Image>` コンポーネントへの置き換えによる最適化が可能か検証し、LCP（Largest Contentful Paint）改善の提案または実装を行う。
- **レスポンシブデザイン**: スマートフォン、タブレット、PCの各ブレイクポイントでレイアウト崩れが発生していないかコードレベルで監査する。
